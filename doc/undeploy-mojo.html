virgo:undeploy

Full name:

net.flybyte.virgo:virgo-maven-plugin:1.0.0:undeploy

Description:

Undeploys an already deployed maven artefact from Eclipse Virgo via the deployer MBean. Check the Virgo sourcecode for more information.

Attributes:

  • Requires a Maven project to be executed.

Required Parameters

Name Type Since Description
virgoRoot File - The root directory of the Virgo installation.

Optional Parameters

Name Type Since Description
osgiVersion String - An OSGi compatible version of the project version. Default value used for transformation is ${project.version}. Once the getter method is called it will transform the first -(hyphen) it can find into a .(dot).
Default value is: ${project.version}.
password String - The password used to connect to Virgo via JMX.
Default value is: springsource.
recoverable boolean - Defineds whether or not the artifact should be recovered after a server restart.
Default value is: true.
serviceUrl String - The service url of the JMX management server.
Default value is: service:jmx:rmi://localhost:9875/jndi/rmi://localhost:9875/jmxrmi.
symbolicName String - The symbolic name of the bundle. In case one is using something different than ${project.groupId}.${project.artifactId} for the symbolic name when using the Maven bundle plugin or Bundlor Maven plugin change the value of this property accordingly.
Default value is: ${project.groupId}.${project.artifactId}.
truststoreLocation File - The location of the truststore used by Virgo. Instead of definining this property in the pom file it is also possible to use a VM argument -Djavax.net.ssl.trustStore when executing the Maven goal. In case this property has neither been set via VM arguments nor in the POM configuration the virgo root directory will be used and a relative path /config/keystore will be added to it.
user String - The username used to connect to Virgo via JMX.
Default value is: admin.

Parameter Details

osgiVersion:

An OSGi compatible version of the project version. Default value used for transformation is ${project.version}. Once the getter method is called it will transform the first -(hyphen) it can find into a .(dot).
  • Type: java.lang.String
  • Required: No
  • Default: ${project.version}

password:

The password used to connect to Virgo via JMX.
  • Type: java.lang.String
  • Required: No
  • Default: springsource

recoverable:

Defineds whether or not the artifact should be recovered after a server restart.
  • Type: boolean
  • Required: No
  • Default: true

serviceUrl:

The service url of the JMX management server.
  • Type: java.lang.String
  • Required: No
  • Default: service:jmx:rmi://localhost:9875/jndi/rmi://localhost:9875/jmxrmi

symbolicName:

The symbolic name of the bundle. In case one is using something different than ${project.groupId}.${project.artifactId} for the symbolic name when using the Maven bundle plugin or Bundlor Maven plugin change the value of this property accordingly.
  • Type: java.lang.String
  • Required: No
  • Default: ${project.groupId}.${project.artifactId}

truststoreLocation:

The location of the truststore used by Virgo. Instead of definining this property in the pom file it is also possible to use a VM argument -Djavax.net.ssl.trustStore when executing the Maven goal. In case this property has neither been set via VM arguments nor in the POM configuration the virgo root directory will be used and a relative path /config/keystore will be added to it.
  • Type: java.io.File
  • Required: No
  • Expression: ${javax.net.ssl.trustStore}

user:

The username used to connect to Virgo via JMX.
  • Type: java.lang.String
  • Required: No
  • Default: admin

virgoRoot:

The root directory of the Virgo installation.
  • Type: java.io.File
  • Required: Yes
  • Expression: ${virgoRoot}
